# TulaBiz

TulaBiz is a fintech application designed to help SMEs and merchants monitor, organize, and grow their financial operations from one place. It combines payment management, customer tracking, product management, QR-based checkout, promotional tools, receipts, notifications, and business analytics into a single merchant-focused experience.

## What TulaBiz helps businesses do

- Track revenue, expenses, balances, and recent transaction activity in real time.
- Accept and manage payments through multiple checkout flows.
- Generate payment links for customers and campaigns.
- Create and share QR codes for in-store or fast checkout.
- Record cash receipts with item details, customer names, and notes.
- Monitor settlement history, wallet movements, and payment status.
- Manage customer records, spending history, and customer notes.
- Track products, stock levels, categories, pricing, and low-stock items.
- Create and monitor promo codes and campaign offers.
- Review business analytics, reports, income trends, expense trends, and budget usage.
- Configure merchant settings for billing, notifications, security, branding, and support.

## Key modules

- Home dashboard with balance visibility, profit summary, invoices, and recent activity.
- Payment links for creating secure customer payment URLs.
- QR checkout for receiving payments through scan-and-pay flows.
- Cash receipts for documenting offline transactions clearly.
- Customers management for storing and reviewing customer profiles.
- Payments screen for tracking settlements and payment history.
- Products and inventory management with detailed product views.
- Promo codes for running discounts and campaign offers.
- Analytics for business performance and financial reporting.
- Setup checklist to guide merchants through the core onboarding steps.
- Help, docs, and settings screens for support, configuration, and account management.

## Getting started

1. Clone the repository.
2. Install dependencies:

   ```bash
   flutter pub get
   ```

3. Run the app:

   ```bash
   flutter run
   ```

## Project structure

- `lib/screens` - main application screens
- `lib/widgets` - reusable UI components
- `lib/models` - data models
- `assets/images` - branding and UI assets

## Notes

- The app is currently focused on merchant experience and financial visibility.
- Some screens are demo-driven or visually staged, so the README reflects the implemented product flows rather than backend-integrated production features.
